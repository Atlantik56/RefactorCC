VeloCore is a Next.js static-export website for a cycling and IT club. The project uses App Router, local visual assets, clean routes, and a GitHub Pages deployment workflow.

## Current Status

- Main branch: `main`
- Repository: `https://github.com/Atlantik56/VeloCore`
- Production: `https://atlantik56.github.io/VeloCore/`
- Deployment: GitHub Actions to GitHub Pages
- Frontend: Next.js App Router static export
- Backend: not included yet; join forms are client-side demos
- Assets: all site images are stored locally in `public/assets`

## Tech Stack

- React 19
- Next.js 16
- Plain CSS
- pnpm
- GitHub Actions
- GitHub Pages

## Pages

The app is statically generated by Next.js with clean routes:

- `/` - landing page
- `/routes` - route catalogue
- `/community` - community page
- `/training-plans` - training plans
- `/training-plans/beginner-indoor` - detailed beginner indoor plan
- `/training-plans/beginner-road` - detailed beginner road plan
- `/training-plans/intermediate-indoor` - detailed intermediate indoor plan
- `/training-plans/intermediate-road` - detailed intermediate road plan
- `/road` - road cycling
- `/track` - track cycling
- `/mtb` - mountain bike
- `/gravel` - gravel
- `/cyclocross` - cyclocross
- `/bmx` - BMX

## Run Locally

```bash
pnpm install
pnpm run dev
```

Then open the local URL printed by Next.js.

## Build

```bash
pnpm run build
```

The production output is written to `out/`.

## Docker

Build and run the static site container:

```bash
docker compose up --build -d
```

The site will be available at `http://localhost:8080/`.

The image builds the Next.js static export in a Node stage and serves `out/` from Nginx.

## Optimization

The production build is configured in `next.config.mjs`:

- static export is enabled with `output: 'export'`
- deploy base path is configurable with `NEXT_PUBLIC_BASE_PATH`
- the GitHub Pages workflow builds with `NEXT_PUBLIC_BASE_PATH=/VeloCore`
- local builds default to root for custom-domain or local hosting checks
- source maps are disabled for production
- Next's default production optimizations are used
- assets are kept local in `public/assets`
- kit showcase images were compressed and normalized to practical display dimensions

Current local image assets are stored in `public/assets`, including Wikimedia-sourced images in `public/assets/wikimedia`.

## GitHub Pages Deploy

Deployment is handled by `.github/workflows/deploy-next.yml`.

On push to `main`, the workflow:

1. installs dependencies with pnpm,
2. builds the Next static export,
3. uploads `out/` as a Pages artifact,
4. deploys it to GitHub Pages.

## Security Notes

- Page HTML is sanitized before insertion into the client shell.
- Internal links inside sanitized page HTML are rewritten through the configured base path.
- A Content Security Policy is emitted from the Next layout.
- Carousel captions use an allowlist sanitizer.
- Circuit animation caches geometry and pauses when the tab is hidden.
